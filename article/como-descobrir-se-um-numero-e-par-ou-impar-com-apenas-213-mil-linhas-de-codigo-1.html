Como descobrir se um número é par ou ímpar com apenas 213 mil linhas de código

Porque usar % 2 se você pode usar 213.000 if/else?

Publicado em : 26 de Junho de 2025 às 14:02

Vinícius Pires

Inspiração

Após assistir ao vídeo "4 bilhões de declarações "If"" do @ThePrime, que é um react de um artigo feito pelo @andreasjhkarlsson, fiquei inspirado e decidi criar algo parecido.

A ideia

A princípio, a ideia era fazer um script em Python com milhares de linhas e que fizesse alguma operação claramente trivial e desnecessária.

Por isso escolhi verificar se um número é ímpar ou par da maneira menos eficiente possível, com um bloco if/else para cada um dos números.

Mão na massa

Para automatizar o processo de criação desse script "meme", criei o create.py.

Esse script é basicamente um forloop que escreve os blocos if/else no arquivo impar-ou-par.py.
Parando apenas quando atinge o breakpoint, ou tamanho esperado em bytes especificado pelo usuário.

O problema

O script create.py funciona perfeitamente, criando arquivos enormes com milhares de linhas de forma eficiente.
Porém...

Ao tentar executar o bendito impar-ou-par.py, o Python acusa um erro.

MemoryError: Parser stack overflowed - Python source too complex to parse

Todos aqueles blocos incontáveis de if/else eram complexos demais para serem parseados pelo interpretador.

Um problema parecido com o que @andreasjhkarlsson também enfrentou na sua jornada.

Solução

Por conta desse empecilho, precisei criar um script run.py que é capaz de rodar o conteúdo gigantesco do impar-ou-par.py.

Inicialmente, faço a validação do arquivo impar-ou-par.py, verificando se ele está lendo o número do usuário, e ao final se ele tem um último bloco else para parar o código.

    headerString = read_line(1).strip()
    footerString = read_line(-2).strip()

    if 'numero = int(input("Digite um número: "))' in headerString and \
       'print(f"Não é possível determinar se {numero} é um número ímpar ou par. :(  ")''' in footerString:

Após essa validação, executo a primeira linha extraída do impar-ou-par.py na thread do run.py

# output
numero = int(input("Digite um número: "))

 try:
     string = read_line(1).strip()
     exec(string ,  globals())
     return True
 except Exception as e:
     print(e)
     return False

E com um simples forloop, procuro o bloco if/else no qual a variável numero cairia, e executo seu conteúdo.

fp = open(file)
    for i, line in enumerate(fp):
        if f'{numero}' in line:
            string = read_line(i+1).strip()
            exec(string)
            exit(0)
fp.close()

Dessa forma, consigo executar o conteúdo do impar-ou-par.py, mantendo o comportamento esperado e de forma eficiente.
