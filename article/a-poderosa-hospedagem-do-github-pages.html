A poderosa hospedagem do Github Pages

Um experimento prático para transformar um projeto dinâmico em páginas estáticas e simplificar o deploy sem abrir mão da estrutura original.

Publicado em : 25 de Junho de 2025 às 19:09

Vinícius Pires

Início

Como um projeto paralelo ao trabalho, decidi codar um simples blog para escrever meus pensamentos, contar sobre alguns projetos pessoais e até experiências profissionais.

Utilizando Django como tecnologia principal, em alguns dias o projeto já tinha saído do papel.

  • Página inicial com os artigos mais recentes

  • Implementação de busca com base nos títulos ou aparição de termos nos artigos

  • Implementação de Markdown para o corpo dos textos

Porém ainda precisava de uma solução cloud para o deploy do projeto.

O problema

Pensando melhor sobre como colocar esse projeto em produção, lembrei do GitHub Pages, que hospeda simples arquivos estáticos HTML, CSS, imagens e etc.

Mas como portar um projeto Django com banco de dados e páginas que são carregadas a partir de templates, para uma hospedagem de arquivos totalmente estáticos?

Solução

Para isso, criei um script build.sh inspirado no processo de build de diversos frameworks.

O script executa o servidor Django, e por meio de wget, salva o index.html na raiz do projeto, bem como os arquivos static, media e os próprios artigos, que antes eram servidos ao user pelo Django.

Salvando todos esses elementos na pasta raiz do projeto, de forma a espelhar as rotas estipuladas no projeto principal do Django, basta apenas commitar que o GitHub Pages realizará o deploy automático.

Dessa forma, a dor de cabeça de configurar uma VPS, com controle de custos, fail2ban e diversos ajustes para evitar invasões foi remediada com um simples script de build.

Conclusão

Nem sempre a solução mais complexa, bonita e elegante será a escolhida.

Nesse período trabalhando com desenvolvimento web e TI, aprendi que, muitas vezes, realizar a tarefa com o menor custo, mantendo os requisitos e projeções mais próximos da realidade, é mais importante do que implementar a solução mais atual do mercado, que às vezes demanda muito mais esforço e traz diversas dores de cabeça que não se traduzem na eficiência da entrega do projeto.
